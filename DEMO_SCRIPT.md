# ReuniteRC 3-Minute Demo Script

## 0:00-0:20 - Opening

ReuniteRC is a digital reunification and lost-and-found solution for major programmes in Redemption City. It strengthens the existing Information Bureau process by turning official camp locations into Reunite Points.

Open `/`.

## 0:20-0:45 - Reunite Point

Open `/reunite-points`.

Show `RP-014` Arena Rear. Point out the ReuniteRC branding, visible Point Code, location name, short URL, QR rendering, fallback instruction and tamper-check guidance.

Key line: if the online form cannot load, the person can still report the Point Code to official staff.

## 0:45-1:25 - Person Flow

Open `/demo`, then `/report/person`.

Show the missing-child and found-child flow from `RP-014`. Open `/matches/person` and highlight the score, tier and transparent rule-based reasons.

Key line: this is not automatic public matching. Human verification is required before reunion.

Open `/handover/person/person_match_demo`. Complete the verified reunion while online. Toggle degraded connectivity and show that final closure is blocked.

## 1:25-1:55 - Item Flow

Open `/matches/item`.

Show the lost-bag and found-bag recommendation. Highlight item category, Reunite Point compatibility, description tag overlap and staff-only hidden detail.

Open `/release/item/item_match_demo`. Complete item release only after proof-of-ownership verification. Toggle degraded connectivity and show release blocking.

## 1:55-2:25 - Offline And PA Fallback

Return to `/demo`. Toggle degraded connectivity and queue an offline report.

Open `/announcements`. Show that PA fallback can be queued for urgent or unresolved cases, but PA does not automatically resolve a person or item case.

## 2:25-3:00 - Leadership Outcome

Open `/dashboard`, then `/analytics`.

Show operational metrics, offline queue count, active Reunite Points and aggregate-only leadership analytics.

Close with the recommendation: add temporary volunteer coverage near Arena Rear during closing periods.
